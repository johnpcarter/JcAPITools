rollbackAPI 
##Rollbacks the last promotion and publication ##

Should only be used after using deployAPI service in order to cancel any changes.
Will first unpublish the API from the given API Portal if provided, then it will rollback the API changes from the target API Gateway as indicated by the given stage.
If the stage is omitted, then API will be unpubished from the given API Gateway, otherwise it will be removed from the API portal associated with the API Gateway associated with the stage!

If the stage is omitted, then the API will only be removed from the associated API Portal.

#INPUT#
apiServer
	apiGatewayURL
	apiCredentials - user/password for gateway separated by ":"
apiName - Name of API to undeploy
apiVersion - Version of API to undeploy (or latest if omitted)

where
	stageName - If provided, used by promotion to rollback the API changes 
	reason - Label for rollback.
	description - Detailed label for rollback.
	portalName: If provided will remove API from given API Portal as determined the immediate API Gateway if stage omitted, alternatively will remove
			the API from the Portal associated with the API Gateway as defined by the stage name.

#OUTPUT#
none
