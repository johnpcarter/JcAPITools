## Publishes API to API Gateway ##

Deploys the swagger definition provided the named API descriptor hosted locally.

If the API already exists, will either update the existing version with the new swagger without affecting any meta-data. Alternatively if 'apiAllowUpdate' is false (default)
will create a new copy of existing API and apply the swagger definition to that.

Caller can also specify a default app, group, maturity and version to be set if creating the API for the first time.

The API will not be activated by default, use the service setAPIAttributes to active it.

#INPUT#
apiGatewayURL
apiCredentials - user/password for gateway separated by ":"
apiAllowAllowUpdate - false by default and creates a new version if API already exists, set to false if you want to update existing API
apiMaturity - default label to be used if creating new API
apiGroup - default group to be used if creating new API
apiVersion - Default version to be used if creating new API
defaultApp - Creates given app of not yet created and associates it with the API
apiEndpoint - replaces api endpoint with this in swagger before being published
apiName - Name of API to deploy

#OUTPUT#
apis - list of the API's after registration
	id - Internal id of API associated with API Gateway
	apiName - name of API as given by swagger
	apiVersion - version either that of swagger, default or next sequence if creating a new version
